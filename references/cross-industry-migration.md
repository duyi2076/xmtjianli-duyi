# 跨行业经历强制重构规则

## 核心原则

**只要用户提供了任何工作经历，就必须从经历中挖掘可迁移到新媒体运营的能力，强制重构成新媒体运营项目。**

不允许因为「经历不相关」就判定为不适用，或让用户补充新媒体经历。

## 可迁移能力映射表

| 原始岗位类型 | 可重构为新媒体运营能力 | 重构方向 |
|-------------|---------------------|---------|
| 销售 / 地推 / 电销 | 私域运营、用户运营、转化文案 | 引流话术、客户分层、社群转化 |
| 行政 / 文员 / 助理 | 公众号运营、图文排版、内部传播 | 内容编辑、活动通知、资料整理 |
| 客服 / 售后 | 用户运营、社群运营、内容 FAQ | 用户分层、常见问题内容化 |
| 教师 / 培训师 | 知识付费内容、短视频口播、课程拆解 | 选题、脚本、知识类账号运营 |
| 设计 / 美工 | 视觉内容、封面设计、图文制作 | 小红书/抖音视觉内容生产 |
| 电商 / 零售 | 直播运营、商品文案、投流素材 | 带货内容、产品卖点提炼 |
| 医疗 / 护理 | 健康科普内容、患者社群、专业图文 | 医疗垂类账号内容运营 |
| 餐饮 / 服务 | 本地生活内容、探店视频、用户评价运营 | 小红书/抖音本地生活运营 |

## 重构原则

### 1. 动作迁移

把原始工作中的具体动作改写成新媒体运营动作：

- 「给客户发微信」→「私域用户触达」
- 「整理资料」→「内容素材库搭建」
- 「发朋友圈宣传」→「朋友圈内容运营」
- 「组织活动」→「线上活动策划执行」
- 「写工作汇报」→「数据复盘与内容优化」

### 2. 数据迁移

原始工作中的数据尽量保留并改写为新媒体指标：

- 「服务 200+ 客户」→「累计触达用户 200+ 人」
- 「月销售额 5 万」→「通过内容/私域转化月 GMV 5 万」
- 「组织 10 场活动」→「累计策划执行 10 场线上活动」
- 「写 50 篇文章/通知」→「累计输出图文内容 50+ 篇」

### 3. 场景重构

把原始工作场景套到新媒体运营场景里：

- 客户沟通 → 用户评论/私信回复
- 产品推介 → 种草内容/带货脚本
- 活动策划 → 线上快闪群/直播活动
- 资料整理 → 选题库/素材库/知识库

### 4. 强制输出

无论原始经历看起来多不相关，最终必须输出至少 3 个项目制经历，且每个项目必须包含新媒体运营动作和数据。

### 5. 标注要求

跨行业重构部分必须在 AI 补充说明中标注：

> 「原经历为 XXX，已按新媒体运营能力做迁移重构」

## 质量红线

- 不得编造用户没有做过的核心动作
- 不得把团队/公司数据直接算到个人头上
- 迁移后的数据必须能在原经历中找到对应依据
- 所有迁移内容必须标注，便于用户面试前核实

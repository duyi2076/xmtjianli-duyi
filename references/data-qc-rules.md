# 数据质检与合理性修正规则

## 核心原则

**不允许把账号/团队历史累计直接算到个人头上。**

## 质检公式

```
个人合理产出上限 = 时间跨度（月） × 岗位合理月均产出
```

## 内容产出基准

| 岗位类型 | 合理月均产出基准 |
|---------|----------------|
| 公众号图文（全职日更） | 60-90 篇/月 |
| 小红书图文/短视频（全职矩阵） | 80-150 条/月 |
| 抖音短视频（全职矩阵） | 80-150 条/月 |
| 私域被动引流 | 100-300 人/月 |
| 快闪群活动 | 2-4 场/月 |

## 年度产出上限约束

- 单个项目的内容产出（图文/短视频/笔记）不建议超过 **600 条/年**
- 若原简历数据超出此上限，按以下规则修正：
  - 有明确月均/日产出的：以「年产 ≤600 条」反推，取合理月均约 50 条，重新计算总量
  - 只有总量的：用实际时间跨度按年均 ≤600 条折算
  - 原文明确写「每日 X 篇/条」导致超限的：改为「每周 X-X 篇/条」或「月均约 X 条」

## 留资 / 引流数据核查（强制）

对简历中所有「加微人数」「留资数」「私信咨询数」「获客数」等数据，**必须**进行合理性核查，禁止直接沿用原文而不验证。

核查方式：
1. 根据项目所属行业（教育/知识付费、医疗、本地生活、家居等）和账号阶段（0-1 冷启动、成长期、成熟期），使用 WebSearch 查询该垂类留资/引流的公开行业基准数据。
2. 将原文数据与行业基准进行比对：
   - **在合理区间内 → 保留原值，但用保守表述呈现**（写「约 X」「X+」「X 人左右」，不写上限极限值）
   - **明显超出行业上限** → 按行业基准合理区间下限或中位数下调，并标注「经行业基准修正」
   - **明显低于行业下限但其他数据夸张** → 检查是否把「咨询数」误写为「加微数」，或引流路径描述是否无法支撑
   - **多账号矩阵的留资数据** → 先按账号数拆分，再判断单账号是否合理
3. 所有修正的留资数据必须标注：`【留资数据经行业基准核查修正，需你确认】`

**保守性原则（关键）：**
- "保守"指**表述方式保守**，不是把合理数据再砍一刀。
- 原文数据已在合理区间内的，**保留原值**，只改表述为「约 X」「X+」等。
- 只有原文数据明显超出行业基准时，才按基准下调。
- 绝对禁止：因为"要保守"就把合理的原简历数据人为压低。

## 处理规则

1. **原文数据在合理范围内** → 保留，仍按「原简历提供」标注。
2. **原文总量明显超出个人合理产出上限**（如 8 个月写出 2500+ 篇）→ 判定为「账号/团队历史累计」，必须拆分为：
   - 个人任期内可验证部分：按「时间 × 频次」重新计算
   - 账号原有基础/团队累计部分：从个人成果中剔除
3. **原文只有月均/日均数据，没有总量** → 按实际时间跨度推算总量，作为简历呈现数据。
4. **原文数据明显偏离行业常识**（如单条视频涨粉转化率异常、变现金额与个人角色不匹配）→ 使用 WebSearch 查询该垂类行业基准，替换为合理数字，并标注「数据经行业基准修正」。
5. **所有修正/替换的数据必须标注：** `【数据经 AI 质检修正，需你确认】`

## 特别注意

- 变现金额（GMV/销售额）必须区分「个人直接业绩」和「所在项目/公司 GMV」。后者在简历中仅作为项目规模参考，不作为个人能力数据。
- 粉丝量、阅读量等账号整体数据，必须区分「个人任期内增长」和「账号历史累计」。

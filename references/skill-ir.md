# Skill IR（中间表示）

## 目标

把 `xmtjianli-duyi` 的能力、约束、流程转换为结构化的 Skill IR，便于评估、迁移和复用。

## 能力清单

| 能力 ID | 能力名称 | 输入 | 输出 | 依赖 |
|---------|---------|------|------|------|
| parse | 简历解析 | 原始简历文本 | 结构化经历数据 | 无 |
| calibrate | 时间线校准 | 结构化经历数据 | 无重叠、无空窗的时间轴 | timeline-rules |
| reconstruct | 项目制重构 | 时间轴 + 经历数据 | 项目列表 | workflow |
| migrate | 跨行业迁移 | 非新媒体经历 | 新媒体运营项目 | cross-industry-migration |
| qc | 数据质检 | 项目数据 | 修正后数据 | data-qc-rules |
| advantage | 个人优势写作 | 项目数据 | 5 条优势 | workflow |
| story | 面试讲述生成 | 项目数据 | 口述版 + 详细版 | oral-story-template |
| export | docx 导出 | 简历数据 + 面试数据 | 两个 docx | scripts, assets |

## 约束清单

- C01: 所有经历结束时间必须在今天之前
- C02: 工作经历串行、无重叠、无空窗
- C03: 每段经历 ≥7 个月
- C04: 年产内容 ≤600 条
- C05: 留资/引流数据需经行业基准核查
- C06: 个人优势数据必须严格加总自项目经历
- C07: AI 补充内容必须标注
- C08: 跨行业重构必须标注迁移依据

## 触发器

- `/xmtjianli-duyi`
- 「帮我改简历」
- 「优化这份简历」
- 「把这份简历改成项目制」

- Markdown 预览
- `{姓名}-{求职意向}-简历-修正版.docx`
